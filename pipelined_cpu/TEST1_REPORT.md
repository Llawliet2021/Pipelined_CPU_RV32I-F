# TEST1 报告 — 基础指令测试 (riscv_ex73_74.txt)

---

## 一、测试概述

本测试使用 RISC-V 汇编程序 `riscv_ex73_74.txt`（共 45 条指令），验证流水线 CPU 的**基础整数指令**功能。

**测试文件：**
- 指令文件：`riscv_ex73_74.txt`（45 条十六进制指令，位于 `pipelined_cpu/` 目录）
- Testbench：`pipelined_cpu_tb.v`（硬编码校验 13 次 store，自带逐周期日志）

**测试结果：13/13 ALL PASSED，最终结果 = 25，耗时 48 个时钟周期**

**验证的指令类型：** 算术运算(add/sub/sll/srl/sra)、立即数运算(addi/slli/srli/srai)、逻辑运算(xor/or/and)、U型(lui/auipc)、B型分支(beq/bne/blt/bge/bltu/bgeu)、S型存储(sb/sw)、I型加载(lbu)、跳转(jal/jalr)

---

## 二、逐条指令详解

> `[PC=...]` 是程序计数器，`InstrD=...` 是 IF/ID 流水线寄存器中的 32 位指令编码。每条指令的汇编助记符均基于指令编码精确解码，与 `riscv_ex73_74_annotated.txt` 中的标注完全一致。

### 阶段0：流水线填充（周期1~3）

这三周期 CPU 处于复位状态 (`reset=1`)，所有流水线寄存器清零，PC=0。

```
[cycle=1] PC=00000000  InstrD=00000000
[cycle=2] PC=00000000  InstrD=00000000  
[cycle=3] PC=00000000  InstrD=00000000
```
`InstrD=00000000` 即 NOP（空操作）。复位期间 CPU 不执行任何指令。

```
[cycle=6] PC=0000000c  InstrD=00300393
```
| 指令 | `addi x7, x0, 3` |
|------|------|
| 解释 | x7 = 0 + 3 = 3 |
| CPU 状态 | x7 = 3 |

```
[cycle=7] PC=00000010  InstrD=fff00793
```

### 阶段2：R型/I型运算（周期8~13）

```
[cycle=8] PC=00000014  InstrD=00314433
```
| 指令 | `xor x8, x2, x3`（R型异或） |
|------|------|
| 解码 | rs1=x2(5), rs2=x3(12), rd=x8 |
| 解释 | x8 = x2 ^ x3 = 5 ^ 12 = 9 |
| CPU 状态 | x8 = 9 |

```
[cycle=9] PC=00000018  InstrD=007114b3
```
| 指令 | `sll x9, x2, x7`（R型逻辑左移） |
|------|------|
| 解码 | rs1=x2(5), rs2=x7(3), rd=x9 |
| 解释 | x9 = x2 << x7 = 5 << 3 = 40（0x28） |
| CPU 状态 | x9 = 40 |

```
[cycle=10] PC=0000001c  InstrD=0024d533
```
| 指令 | `srl x10, x9, x2`（R型逻辑右移） |
|------|------|
| 解码 | rs1=x9(40), rs2=x2(5), rd=x10 |
| 解释 | x10 = x9 >> x2 = 40 >> 5 = 1 |
| CPU 状态 | x10 = 1 |

```
[cycle=11] PC=00000020  InstrD=4027d5b3
```
| 指令 | `sra x11, x15, x2`（R型算术右移） |
|------|------|
| 解码 | rs1=x15(-1), rs2=x2(5), funct7[30]=1 即 sra |
| 解释 | x11 = -1 >>> 5 = -1（算术右移用符号位填充，-1 的所有位都是 1） |
| CPU 状态 | x11 = 0xFFFFFFFF = -1 |

```
[cycle=12] PC=00000024  InstrD=00211613
```
| 指令 | `slli x12, x2, 2`（I型逻辑左移立即数） |
|------|------|
| 解码 | rs1=x2(5), imm[4:0]=2, funct3=001, rd=x12 |
| 解释 | x12 = x2 << 2 = 5 << 2 = 20（0x14） |
| CPU 状态 | x12 = 20 |

```
[cycle=13] PC=00000028  InstrD=4047d693
```
| 指令 | `srai x13, x15, 4`（I型算术右移立即数） |
|------|------|
| 解码 | rs1=x15(-1), imm[4:0]=4, funct3=101, imm[11:5]=0100000(srai) |
| 解释 | x13 = -1 >>> 4 = -1 |
| CPU 状态 | x13 = -1 |

### 阶段3：U型指令（周期14~15）

```
[cycle=14] PC=0000002c  InstrD=12345737
```
| 指令 | `lui x14, 0x12345`（U型加载上位立即数） |
|------|------|
| 解释 | x14 = 0x12345 << 12 = 0x12345000 |
| **注意** | LUI 不经过 ALU！立即数通过 WB 阶段的 ImmExt 直接写回（ResultSrc=11） |
| CPU 状态 | x14 = 0x12345000 |

```
[cycle=15] PC=00000030  InstrD=00000817
```
| 指令 | `auipc x16, 0`（U型加 PC 立即数，imm=0） |
|------|------|
| 解释 | x16 = IFID_PC + 0 = 0x2C（AUIPC 指令在 ID 阶段的 IFID_PC 值） |
| **注意** | AUIPC 的 ALU A 操作数来自 PC（ALUSrcA=1），而非 rs1 |
| CPU 状态 | x16 = 0x2C = 44 |

### 阶段5：分支指令（周期26~33）

```
[cycle=26] PC=0000005c  InstrD=00314463
```
| 指令 | `blt x8, x3, +8`（解码: rs1=x8(9), rs2=x3(12), funct3=100→blt） |
|------|------|
| 解释 | 9 < 12（有符号比较）成立 → 跳转到 PC+0x8 = 0x64 |
| 流水线动作 | PCSrcD=01 → FlushD=1 → IF/ID 清零（误取的 0x60 处指令被 NOP 化） |

```
[cycle=27] PC=00000060  InstrD=00000000  （被 FlushD 刷新的 NOP）
```

```
[cycle=28] PC=00000064  InstrD=00316463
```
| 指令 | `bne x8, x8, +8`（rs1=x8(9), rs2=x8(9), funct3=001→bne） |
|------|------|
| 解释 | 9 ≠ 9 不成立 → 不跳转 |

```
[cycle=30] PC=0000006c  InstrD=0021d463
```
| 指令 | `bge x3, x2, +8`（rs1=x3(12), rs2=x2(5), funct3=101→bge） |
|------|------|
| 解释 | 12 ≥ 5 成立 → 跳转 |

```
[cycle=32] PC=00000074  InstrD=0027f463
```
| 指令 | `bgeu x15, x2, +8`（rs1=x15(-1), rs2=x2(5), funct3=111→bgeu） |
|------|------|
| 解释 | 0xFFFFFFFF ≥ 5（无符号比较）→ 跳转 |

### 阶段6：sb 字节存储 + lbu 字节加载（周期34~36）

```
[cycle=34] PC=0000007c  InstrD=00f000a3
```
| 指令 | `sb x15, 1(x0)`（S型字节存储：rs2[24:20]=x15, rs1=x0, imm=1） |
|------|------|
| 解释 | 把 0xFF（x15=-1 的低字节）写入内存地址 0+1=1 |
| dmem 状态 | 地址 1（RAM[0] 的 byte 1）= 0xFF |

```
[cycle=35] PC=00000080  InstrD=00104903
```
| 指令 | `lbu x18, 1(x0)`（I型无符号字节加载：rs1=x0, imm=1, rd=x18, funct3=100） |
|------|------|
| 解释 | x18 = 零扩展(mem[1]) = 0x000000FF = 255 |

> **注：** 下一条指令是 auipc x19, 0，不依赖 x18，因此无需 Load-Use 停顿。

```
[cycle=36] PC=00000084  InstrD=00000997
```
| 指令 | `auipc x19, 0` |
|------|------|
| 结果 | x19 = IFID_PC（指令自身地址） |

### 阶段8：最终循环与结束（周期46~48）

```
[cycle=46] PC=000000b0  InstrD=06202223
```
| 指令 | `sw x2, 100(x0)`（把 x2 = 25 存入地址 100 = 0x64） |

## 三、测试输出日志格式

```
[cycle=N] PC=XXXXXXXX  InstrD=XXXXXXXX  MemWrite=B  DataAdr=XXXXXXXX  WriteData=XXXXXXXX
```

| 列名 | 含义 |
|------|------|
| `cycle` | 当前时钟周期编号（从 1 开始） |
| `PC` | 程序计数器 PC_reg，当前 IF 阶段取指的地址 |
| `InstrD` | IF/ID 流水线寄存器中的 32 位指令编码 |
| `MemWrite` | 0 = 不写内存，1 = MEM 阶段正在写内存 |
| `DataAdr` | 写内存的地址（= ALUResultM） |
| `WriteData` | 写入内存的数据（= WriteDataM） |

当 `MemWrite=1` 时，testbench 执行 `check_write` 对比期望值：

```
PASS @ write_idx=0: addr=00000060, data=00000009
```
- `write_idx=0`：第 0 次 store（从 0 开始计数）
- `addr=00000060`：写入地址是 96（十进制）
- `data=00000009`：写入值是 9（十六进制）

## 四、本测试验证了哪些功能

| 类别 | 具体验证内容 | 涉及指令 |
|------|------------|---------|
| **整数寄存器读写** | 寄存器 x0 恒为 0，x2~x30 可正确读写 | 所有 R/I 型 |
| **ALU 运算** | 加、减、异或、与、或、左移、逻辑右移、算术右移 | add/sub/xor/or/and/sll/srl/sra |
| **立即数生成** | I型(12bit有符号)、S型、B型、U型(20bit)、J型 立即数扩展 | addi/lui/auipc/jal/sb |
| **分支判断** | 相等/不等、小于/大于等于（有符号和无符号） | beq/bne/blt/bge/bltu/bgeu |
| **分支预测** | 预测不跳转 + 预测错误时 FlushD 刷新 IF/ID | 所有 B 型分支 |
| **U型指令** | LUI 立即数直通（不经 ALU）、AUIPC 使用 PC 作为操作数 | lui/auipc |
| **字节存储** | sb 写入单字节到 dmem | sb |
| **字节加载** | lbu 零扩展加载单字节 | lbu |
| **跳转链接** | JAL/JALR 的返回地址 = PC+4 | jal/jalr |
| **流水线执行** | 48 周期完成 45 条指令，CPI ≈ 1.07 | — |
| **最终结果** | x2 = 25，验证程序整体正确性 | — |
