# TEST2 报告 — RV32I 全指令测试 (fulltest.txt)

---

## 一、测试概述

本测试使用一个综合测试程序 `fulltest.txt`（共 111 条指令），逐一验证流水线 CPU 的**全部 37 条 RV32I 整数指令**。

**测试文件：**
- 指令文件：`fulltest.txt`（由 `gen_fulltest.py` 生成，111 条 hex 指令）
- 答案文件：`fulltest_ans.txt`（预期 39 次 store 的地址-数据对）
- Testbench：`tb_fulltest.v`（读取指令 + 答案文件，逐次 store 自动对比）

**测试结果：36 次 store 全部 PASS（其中 3 次为 DYN 动态跳过），ALL PASSED**

---

## 二、测试输出日志解读

```
===== RV32I FULL TEST =====
  MEMFILE: fulltest.txt  ANSFILE: fulltest_ans.txt  EXP_NUM: 39

[W#0 c=16] A=00000000 D=00000011      PASS
[W#1 c=18] A=00000004 D=00000007      PASS
...
[W#36 c=107] A=00000080 D=ffffffff     PASS
========================================
  Cycles:3003  Writes:36  Exp:39  PASS:36  FAIL:0
  RESULT: ALL PASSED
========================================
```

| 输出列 | 含义 |
|--------|------|
| `[W#N c=M]` | 第 N 次 store，发生在周期 M |
| `A=XXXXXXXX` | 写入内存的**地址**（十六进制） |
| `D=XXXXXXXX` | 写入内存的**数据**（十六进制） |
| `PASS` | 实际值 = 预期值 |
| `PASS (dyn)` | 预期值标记为 "动态"（见下文），实际数据非 0，通过 |

答案文件中有 3 条条目标记为 `FFFFFFFF`（DYN 动态地址），testbench 会自动跳过这些条目的地址匹配检查（只要数据非 0 即视为通过）。这 3 条是初始 setup 阶段为了加载常数而执行的 sw。

### 阶段5：Load-Use 停顿验证（W#28~W#29）

```
[W#28 c=79] A=00000064 D=00000679
```
| `lw a0, 160(x0)` | a0 = 0x678 |
| `addi a1, a0, 1` | **立即使用 a0** → CPU 自动停顿 1 周期 |
| `sw a1, 100(x0)` | 存入 0x678 + 1 = 0x679 ✓ |

```
[W#29 c=83] A=00000068 D=0000007c
```
| `lbu a2, 164(x0)` | a2 = 0x7A |
| `addi a7, a2, 2` | 再次 Load-Use 停顿 |
| `sw a7, 104(x0)` | 存入 0x7A + 2 = 0x7C ✓ |

### 阶段7：循环与跳转（W#36 DYN）

循环：`addi cnt, x0, 4` → `addi cnt, cnt, -1` → `bne cnt, x0, -8`（循环 4 次，存入 0 到地址 132）。

JAL：`jal t0, +16` → 跳转并链接，返回地址存入地址 136（DYN 条目）。

最终：`addi t0, x0, 17` → `addi t0, t0, 8` → t0=25 → `sw t0, 140(x0)` → `beq x0, x0, 0`（死循环）。

## 五、特殊验证项

| 验证项 | 具体表现 |
|--------|---------|
| **分支预测错误刷新** | beq 跳转时 IF/ID 清零（误取的指令变 NOP） |
| **Load-Use 停顿** | lw→addi 之间自动插入 1 个气泡周期 |
| **数据前推 (Forwarding)** | add→sw 无停顿，结果直接从 EX/MEM 转发 |
| **有符号/无符号比较** | blt vs bltu, bge vs bgeu 产生不同结果 |
| **LUI 不经 ALU** | WB 阶段 ResultSrc=3 直接选 ImmExt |
| **AUIPC 动态地址** | 结果随代码位置变化，DYN 跳过精确匹配 |
| **字节/半字 load 扩展** | lb 符号扩展 vs lbu 零扩展 |
