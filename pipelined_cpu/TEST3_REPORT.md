# TEST3 报告 — RV32F 单精度浮点指令测试 (ftest_instructions.txt)

---

## 一、测试概述

本测试使用一个综合浮点测试程序（共 76 条指令），验证流水线 CPU 的**全部 26 条 RV32F 单精度浮点指令**。

**测试文件：**
- 指令文件：`ftest_instructions.txt`（由 `gen_fpu_test.py` 生成，76 条 hex 指令）
- 答案文件：`ftest_ans.txt`（预期 27 次 store 的地址-数据对）
- Testbench：`tb_fptest.v`（读取指令 + 答案文件，逐次 store 自动对比）

**测试结果：27/27 ALL PASSED**

|--------|------|
| `[W#N c=M]` | 第 N 次 store，发生在第 M 个时钟周期 |

| `D=XXXXXXXX` | 写入内存的 32 位数据（可能是整数或 IEEE 754 浮点编码） |
| `PASS (dyn)` | "动态条目" — 跳过精确地址匹配（初始 setup 阶段的 sw） |
| `PASS` | 数据值与预期完全匹配 |

## 三、逐阶段详解

### 阶段0：加载浮点常数到内存（W#0~W#4，DYN）

在测试浮点指令之前，需要先把测试用的浮点数加载到 CPU 的数据内存中。因为 RISC-V 的立即数指令（addi）只能编码 12 位值，而要表示 `0x3F800000`（32 位）需要分两步：

```
lui x5, 0x3F800     → x5 = 0x3F800 << 12 = 0x3F800000（浮点 1.0 的编码）
addi x5, x5, 0      → 低 12 位 = 0
sw t0, 200(x0)      → 把 0x3F800000 存入内存地址 200
```

| W# | 周期 | 地址 | 写入的 32 位编码 | 代表的浮点值 | 解释 |
|----|------|------|----------------|------------|------|
| 0 | 8 | 0xC8=200 | `3F800000` | **1.0** | 后续测试的基准值 |
| 1 | 11 | 0xCC=204 | `40400000` | **3.0** | 加法/乘法/除法的操作数 |
| 2 | 14 | 0xD0=208 | `40C00000` | **6.0** | 乘法的操作数 |
| 3 | 17 | 0xD4=212 | `C0A00000` | **-5.0** | 负数的符号注入测试 |
| 4 | 20 | 0xD8=216 | `40000000` | **2.0** | 除法的操作数 |

---

### 阶段1：浮点 Load（将内存值加载到 FP 寄存器）

```
flw f1, 200(x0) → f1 = 1.0    （FP 寄存器 1）
flw f2, 204(x0) → f2 = 3.0    （FP 寄存器 2）
flw f3, 208(x0) → f3 = 6.0    （FP 寄存器 3）
flw f4, 212(x0) → f4 = -5.0   （FP 寄存器 4）
flw f5, 216(x0) → f5 = 2.0    （FP 寄存器 5）
```

> **flw ：** `flw f1, 200(x0)` 把内存地址 200 处的 32 位数据加载到浮点寄存器 f1。与整数 `lw` 不同，flw 写入的是 FP 寄存器堆而非整数寄存器堆。

---

### 阶段2：浮点算术运算（W#5~W#9）

```
[W#5 c=27] A=000000dc D=40800000      PASS
```
| 指令 | 含义 | 计算过程 |
|------|------|---------|
| `fadd f6, f1, f2` | f6 = f1 + f2 | **1.0 + 3.0 = 4.0** → 编码 `40800000` |
| `fsw f6, 220(x0)` | 存入内存地址 220 | 验证 fadd 结果 ✓ |

```
[W#6 c=29] A=000000e0 D=40000000      PASS
```
| `fsub f7, f2, f1` | f7 = f2 - f1 | **3.0 - 1.0 = 2.0** → 编码 `40000000` ✓ |

```
[W#7 c=31] A=000000e4 D=41900000      PASS
```
| `fmul f8, f2, f3` | f8 = f2 × f3 | **3.0 × 6.0 = 18.0** → 编码 `41900000` ✓ |

```
[W#8 c=33] A=000000e8 D=40000000      PASS
```
| `fdiv f9, f3, f2` | f9 = f3 ÷ f2 | **6.0 ÷ 3.0 = 2.0** → 编码 `40000000` ✓ |

```
[W#9 c=35] A=000000ec D=40000000      PASS
```
| `fsqrt f10, f6` | f10 = √f6 | **√4.0 = 2.0** → 编码 `40000000` ✓ |

---

### 阶段3：符号注入指令（W#10~W#12）

这三条指令操作浮点数的**符号位**而不改变数值：

```
[W#10 c=37] A=000000f0 D=bf800000      PASS
```
| `fsgnj f11, f1, f4` | f11 = f1 的数值 + f4 的符号 | 1.0 的数值 + (-5.0) 的符号位 = **-1.0** → `BF800000` |

```
[W#11 c=39] A=000000f4 D=3f800000      PASS
```
| `fsgnjn f12, f1, f4` | f11 = f1 的数值 + **取反** f4 的符号 | 1.0 的数值 + 取反 (-5.0) 的符号 = 1.0 的数值 + (+) = **+1.0** → `3F800000` |

```
[W#12 c=41] A=000000f8 D=bf800000      PASS
```
| `fsgnjx f13, f1, f4` | f11 = f1 的数值 + **异或** f4 的符号 | 1.0 的符号(+) ⊕ (-5.0) 的符号(-) = **-1.0** → `BF800000` |

---

### 阶段4：最小值/最大值（W#13~W#14）

```
[W#13 c=43] A=000000fc D=3f800000      PASS
```
| `fmin f14, f1, f2` | f14 = min(f1, f2) | min(1.0, 3.0) = **1.0** ✓ |

```
[W#14 c=45] A=00000100 D=3f800000      PASS
```
| `fmax f15, f1, f4` | f15 = max(f1, f4) | max(1.0, -5.0) = **1.0** ✓ |

---

### 阶段5：浮点比较指令（W#15~W#17）

**注意：** 浮点比较指令的结果是 0 或 1 的**整数**，要写入**整数**寄存器堆，而不是浮点寄存器堆！

```
[W#15 c=47] A=00000104 D=00000001      PASS
```
| `feq x10, f1, f1` | x10 = (f1 == f1) | 1.0 == 1.0 → **真 = 1** ✓ |

```
[W#16 c=49] A=00000108 D=00000001      PASS
```
| `flt x11, f1, f2` | x11 = (f1 < f2) | 1.0 < 3.0 → **真 = 1** ✓ |

```
[W#17 c=51] A=0000010c D=00000001      PASS
```
| `fle x12, f1, f2` | x12 = (f1 <= f2) | 1.0 <= 3.0 → **真 = 1** ✓ |

---

### 阶段6：类型转换指令（W#18~W#20）

```
[W#18 c=53] A=00000110 D=00000004      PASS
```
| `fcvt.w.s x13, f6` | **浮点→有符号整数** | f6=4.0 → 向零舍入为整数 4 ✓ |

```
[W#19 c=55] A=00000114 D=00000003      PASS
```
| `fcvt.wu.s x14, f2` | **浮点→无符号整数** | f2=3.0 → 无符号整数 3 ✓ |

```
[W#20 c=59] A=00000118 D=40e00000      PASS
```
| `fcvt.s.w f16, t0` | **整数→浮点** | t0=7 → 7.0 → 编码 `40E00000` ✓ |

> **fcvt.s.w ：** 这条指令的 FPU A 端口需要读取**整数寄存器**（t0=7），而不是浮点寄存器。CPU 通过 `FPUSrcInt` 信号来控制这个二选一。

---

### 阶段7：浮点-整数寄存器数据移动（W#21~W#22）

```
[W#21 c=63] A=0000011c D=41200000      PASS
```
| `fmv.w.x f17, t0` | f17 = int(t0) 按位复制 | t0=0x41200000 → f17=0x41200000（即 float 10.0）✓ |

```
[W#22 c=65] A=00000120 D=41200000      PASS
```
| `fmv.x.w x15, f17` | x15 = float(f17) 按位复制 | f17=0x41200000 → x15=0x41200000 ✓ |

> **fmv ：** 它们只是把 32 位数据从一条总线搬到另一条，不做任何数值转换，在需要把浮点数的位模式传给整数寄存器时非常有用。

---

### 阶段8：融合乘加指令（W#23~W#25）

融合乘加（FMA）是 **R4 型指令**——需要 4 个寄存器操作数（rs1, rs2, rs3, rd），并且**在单次操作中完成乘法和加法**，精度损失比分开的 mul+add 更小。

```
[W#23 c=67] A=00000124 D=41100000      PASS
```
| `fmadd f18, f1, f2, f3` | f18 = f1×f2 + f3 | **1.0×3.0 + 6.0 = 9.0** → 编码 `41100000` ✓ |

```
[W#24 c=69] A=00000128 D=41880000      PASS
```
| `fmsub f19, f3, f2, f1` | f19 = f3×f2 - f1 | **6.0×3.0 - 1.0 = 17.0** → 编码 `41880000` ✓ |

```
[W#25 c=71] A=0000012c D=c0e00000      PASS
```
| `fnmsub f20, f3, f5, f4` | f20 = -(f3×f5 - f4) | **-(6.0×2.0 - (-5.0)) = -(12+5) = -7.0** → 编码 `C0E00000` ✓ |


---

### 阶段9：最终验证（W#26）

```
[W#26 c=77] A=00000130 D=40800000      PASS
```
| 指令序列 | 含义 |
|---------|------|
| `flw f1, 200(x0)` | 重新加载 f1=1.0 |
| `flw f2, 204(x0)` | 重新加载 f2=3.0 |
| `nop` / `nop` | 两条空操作（给 flw 结果留出写回时间） |
| `fadd f21, f1, f2` | f21 = 1.0 + 3.0 = 4.0 |
| `fsw f21, 304(x0)` | 存入地址 304，编码 `40800000` |

## 六、特殊验证项

| 验证项 | 具体体现 | 对应的测试条目 |
|--------|---------|-------------|
| **IEEE 754 编码** | 所有浮点结果用 hex 编码正确 | W#5~W#26 |
| **fsw 数据来自浮点寄存器** | fsw 通路走 FPMemSrc=1，选择 FSrcB_fwd | 所有 fsw 条目 |
| **FP A 端口转发** | fmv.x.w 从刚写入的 f17 读取 | W#22 |
| **FP B 端口转发** | 连续 FP 运算间无需 NOP | W#5~W#9（fadd→fsub→fmul→fdiv→fsqrt 每拍一条） |
| **FPU 整数结果转发** | feq/flt/fle 结果经 ResultM 转发至 sw | W#15~W#17 |
| **FPUSrcInt 选择整数源** | fcvt.s.w 和 fmv.w.x 从整数 rs1 读取 | W#20~W#21 |
| **R4 三读端口** | fmadd 同时读 f1(fs1) + f2(fs2) + f3(fs3) | W#23~W#25 |
| **FSQRT 负输入检测** | 未触发（f6=4.0 为正数） | W#9 |
| **Load-Use stall gap** | flw→fadd 之间有 2 条 NOP | W#26 |
